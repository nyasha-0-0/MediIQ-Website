# 🏥 MediIQ – Intelligent Patient Triage and Queue Management System

## 📌 Overview
MediIQ is a web-based system designed to assist trained medical staff in triaging and managing patients in emergency settings. It helps in registering patients, analyzing symptoms, and prioritizing them in a queue based on urgency.

## ⚙️ Core Idea
The system combines AI-assisted decision-making with human supervision to ensure both efficiency and safety. It is intended to be used by medical professionals, where AI provides recommendations but final decisions remain under expert control.

## 🧠 Features
- 📋 Patient registration and queue management  
- ⚠️ Triage-based prioritization using multiple medical standards (CTS, ATS, MTS)  
- ⏱️ Estimated maximum waiting time for patients  
- 🩺 Symptom selection and basic test recommendations  
- 👨‍⚕️ Doctor availability tracking  
- 🧭 Navigation assistance within the hospital  
- 🆔 ABHA ID integration for maintaining and accessing patient records  

## 🛠️ Tech Stack
- 🌐 Frontend: HTML, CSS, JavaScript, TypeScript  
- ⚙️ Backend: Node.js  
- 🔗 APIs and database integration (for patient data and system flow)  

## 🚧 Challenges
- Designing a unified triage system by combining Canadian Triage System (CTS), Australian Triage Scale (ATS), and Manchester Triage System (MTS)  
- Translating medical guidelines into a structured and safe decision system  
- Handling edge cases where incorrect prioritization could impact patient safety  
- Building the entire system from scratch without an existing unified framework  

## 🚀 Status
⚠️ Under Active Development  
This project is currently in the research and testing phase.

## 🔒 Note

## 🧠 Learnings
- Real-world systems require balancing accuracy, safety, and usability  
- AI systems in healthcare must always include human oversight
